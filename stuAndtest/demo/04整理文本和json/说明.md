- 1.文件结构: ==一个角色一个文件夹，拖到脚本同级文件夹下即可,地址`AppService/src/console/addCharacter/`==
    ```js
    - character_1 
      - photos // photos类型图片
        - a.png
        - b.jpeg ...
      - c_cover.png // cover
      - d_avatar.jpg // avatar
      - character.txt // Chatgpt对于这个角色的描述文本
    ```
    > 支持[".jpg", ".jpeg", ".png", ".txt"]
- 2.生成结果： 
  ```js
    -> public/ 
      + character_avacov/  // cover和avatar图片集合
      + character_photos/  // photos图片集合

    // 同理目录
      + character_result.json  // 可插入character集合的数据
  ```
- 3.脚本文件功能：
  ```js
    - parseTxtToJson.js: 把对应的txt文件转化为json数据
    - handleJson.mjs: 整理数据生成json文件和分类图片
  ```
- 4.==执行命令==: 在`handleJson.mjs`的父文件夹终端下执行  
  ```bash
    node handleJson.mjs
  ```
- 5.注意事项
  - ==每次执行完一次脚本后，删除掉所有的角色文件夹，下次执行拖入新的角色文件夹，否则会重复读取上次剩余的角色信息，造成角色冗余==
  - ==同时每次执行完成后，保存character_result.json，下次会被覆盖==
